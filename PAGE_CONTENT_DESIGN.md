# 页面内容设计清单

本文档详细列出了CPU漏洞检测平台各页面的内容元素，供设计和修改页面时参考。


## 一、布局通用元素

### 1.1 顶部导航栏
| 元素 | 说明 | 位置 |
|-----|------|-----|
| 平台Logo | 显示平台名称/图标 | 左侧 |
| 导航菜单 | 数据大屏、POC、EXP、代码检测 | 顶部中央 |
| 系统信息选择器 | CPU型号、操作系统选择 | 右侧 |
| 用户头像 | 预留用户区域 | 右侧 |

### 1.2 侧边栏
| 元素 | 说明 | 状态 |
|-----|------|-----|
| 快捷导航 | 快速跳转到各页面 | 始终显示 |
| 统计信息 | 显示总漏洞数等 | 可折叠 |

## 三、POC列表页面 (POCView)

### 3.1 搜索筛选区域
```
搜索框:
├── 占位符: 搜索漏洞名称...
├── 搜索字段: name, attackName
└── 实时过滤

筛选条件:
├── CVE类型下拉:
│   ├── 全部
│   ├── 瞬态执行漏洞
│   ├── 预测执行漏洞
│   ├── 缓存漏洞
│   └── 侧信道漏洞
├── 攻击类型下拉:
│   ├── 全部
│   ├── Spectre类攻击
│   ├── Meltdown类攻击
│   └── ...
└── 风险等级:
    ├── 全部
    ├── 高危
    ├── 中危
    └── 低危
```

### 3.2 漏洞卡片列表
```
每个卡片包含:
├── 漏洞名称 + 图标
├── CVE类型标签
├── 攻击类型标签
├── 风险等级标签(高危=红/中危=黄/低危=绿)
├── 成功率: XX%
├── 平均耗时: XXms
├── 支持的CPU型号(省略显示)
├── 操作系统支持图标(Linux/Windows/macOS)
└── 操作按钮:
    ├── 查看详情
    └── 下载POC
```


## 四、EXP列表页面 (EXPView)

### 4.1 漏洞列表
```
与POC页面类似，但显示内容侧重:
├── 漏洞名称
├── CVE类型
├── 攻击类型
├── 风险等级
├── 成功率
├── 攻击者代码预览
└── 受害者代码预览
```

### 4.2 代码展示区域
```
Tab切换:
├── 攻击者代码
└── 受害者代码

代码显示:
├── 语法高亮
├── 行号显示
└── 代码语言: C/ASM

操作按钮:
├── 复制代码
└── 下载代码
```


## 五、代码检测页面 (DetectView)

### 5.1 文件上传区域
```
拖拽上传区域:
├── 提示文字: 拖拽代码文件到此处或点击上传
├── 支持格式: .c, .cpp, .h, .py, .js等
└── 限制: 仅支持单个文件

或点击上传按钮
```

### 5.2 分析流程动画
```
显示步骤:
├── 📥 上传代码
├── 🔍 静态分析
├── 🎯 漏洞匹配
├── ⏱️ 风险评估
└── 📄 报告生成

动画效果: 步骤依次高亮
```

### 5.3 检测结果展示
```
检测到漏洞时显示:
├── 漏洞名称
├── CVE类型
├── 攻击类型
├── 风险等级
├── 详细描述
└── 漏洞位置(行号)

无漏洞时显示:
└── "未检测到已知漏洞"
```

### 5.4 EXP生成区域
```
生成按钮: 根据漏洞类型生成攻击代码
生成后显示:
├── 攻击者代码预览
├── 受害者代码示例
└── 下载EXP按钮
```


## 六、漏洞详情页面 (VulnDetail)

### 6.1 基本信息区
```
漏洞名称 (大字)
├── CVE编号
├── CVE类型
├── 攻击类型
├── 风险等级(高危/中危/低危)
├── 漏洞描述(多行)
└── 漏洞标签 []
```

### 6.2 技术详情
```
受影响的CPU型号列表:
├── Intel系列
├── AMD系列
└── ARM系列

支持的操作系统:
├── Linux ✓
├── Windows ✓
└── macOS ✓

攻击成功率: XX%
平均攻击时间: XXms
```

### 6.3 POC代码区
```
标题: POC代码
代码类型: C/ASM
功能: 展示漏洞验证代码
操作:
├── 复制代码
└── 下载POC
```

### 6.4 EXP代码区
```
Tab切换:
├── 攻击者代码
└── 受害者代码

每个Tab包含:
├── 代码语法高亮
├── 复制按钮
└── 下载按钮
```

### 6.5 相关信息
```
相关漏洞链接
参考文献/CVE链接
```
